Chapter 4

Cryptographic hash functions and mining

Most of the internet security is nowadays based on cryptographic hash functions like SHA256.

What are hash functions?

Hash functions are mathematical functions that takes a message in order to generate a hash (commonly a 256 bit binary word).

For example:

SHA256("message") = 00101001010010011101...011011

Like the digital signature, the only way to find back a message from the output of a hash function is to guess it.

About cryptographic hash functions:
    - They are deterministic: The same message always result in the same hash
    - It is easy to compute a hash from a word.
    - A small change in the message completely changes the hash.
    - It is impossible to find two different messages with the same hash.

How do we use it for Coinet?

In our case, we select a number N between 1 and 256 for example 40. Then, we try to find a hash with forty 0 at the begining.

ex: 00000000000000000000000000000000000000010101010101010101010111010111110110111...

To find this hash, we use the history of the transactions and a number that we should guess.

We should find a number so this equality is true:

SHA256( history of transaction + number ) = hash with N 0's at the beginning

So we have to try several numbers and guess it, this step is called mining. The probability to find a good hash is 1/(2^N). Which means that for N=40, the probability of having a good hash is 1/(2^40)= 1/ 1 099 511 627 776. N changes periodically so it takes around 10 minutes to find a good number.

This particular number that we just found is called a proof of work. It proves that the miner had to make a lot of computational work and it has to be easy to verify.

The most important step about understanding Coinet is to understand why people have to keep mining and why it is an awesome idea in order to securise our currency. These questions will be answered in our next chapter: the blockchain.
